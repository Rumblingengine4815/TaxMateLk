## THE PROBLEM

The Inland Revenue (Amendment) Act No. 11 of 2026 (certified June 3, 2026) expanded 
Sri Lanka's 5% Withholding Tax (WHT) to 25+ new professions for the first time, 
including: IT specialists, social media managers, photographers, translators, writers, 
coaches, musicians, videographers, beauticians, and event organisers.

Thousands of Sri Lankan freelancers now have legal tax obligations they don't know about.
The IRD portal is broken and confusing. Generic AI tools (ChatGPT, Gemini) have training 
cutoffs that predate the June 2026 amendment and cannot answer these questions accurately.
There is no existing conversational tool that answers Sri Lankan tax questions grounded in 
official IRD documents with citations.

## AGENT PIPELINE (4 VISIBLE STEPS — ALL REQUIRED FOR GOLD)

```
User question
    ↓
Step 1 — CLASSIFY (Groq LLM call)
    Determines: INCOME_TYPE | WHT | QUARTERLY | CALCULATION | GENERAL
    ↓
Step 2 — RETRIEVE (ChromaDB semantic search)
    Pulls 4 most relevant chunks from the 12 IRD documents
    ↓
Step 3 — CALCULATE (Pure Python tool — NO LLM)
    If WHT question → calculate_wht(amount)
    If quarterly question → get_quarterly_schedule()
    If tax calculation → calculate_tax(income, type, ...)
    ↓
Step 4 — GENERATE + CITE (Groq LLM call)
    Combines retrieved context + tool output → answer with [Source] [Confidence]
```

## TECHNIQUES (ALL MUST EARN THEIR PLACE FOR GOLD)

**RAG:** Needed because the June 2026 amendment postdates all model training cutoffs. 
Every answer must be grounded in real IRD documents, not hallucinated rules.

**Agents + Tool Calling:** Needed because answering "how much WHT will my client deduct?" 
requires classification → retrieval → calculation → citation — a genuinely multi-step 
workflow a single LLM call cannot do accurately.

**QLoRA Fine-Tuning:** Fine-tunes llama-3.1-8b on 80 Sri Lankan tax Q&A pairs to improve 
citation formatting and amendment-specific accuracy. Evaluated against base model on a 
held-out 10-question set to prove measurable improvement.

**Calculation Tool:** Pure Python — prevents LLM arithmetic errors on tax band calculations. 
The LLM decides WHEN to call it and WITH WHAT parameters; the tool does the math.
